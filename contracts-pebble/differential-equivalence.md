# Differential Equivalence: Aiken vs Pebble pharma_trace

Reproduce: `cd contracts-pebble && node scripts/diff-equiv.mjs`

Both validators (`contracts/validators/pharma_trace.ak` and
`contracts-pebble/src/index.pebble`) implement the counter-pattern state
machine. The harness builds Plutus V3 ScriptContexts by hand, evaluates
both compiled scripts against the same context with
`@harmoniclabs/plutus-machine`, and compares accept/reject outcomes.

Not a proof of equivalence (exhaustive/symbolic testing would be required)
— a curated set of representative scenarios per endpoint, including
regression coverage for each known Pebble bug (see
[`pebble-known-bugs.md`](./pebble-known-bugs.md)).

## Setup

- **Aiken script**: `contracts/plutus.json` (blueprint generated by
  `aiken build`).
- **Pebble script**: `contracts-pebble/out/script.hex` (built with stock
  `@harmoniclabs/pebble` 0.1.8 via `npm run build`).
- **Params**: `manufacturer` VKH + `seed` `OutputReference`, applied
  identically to both.
- **Redeemer Constr indices**: aligned across both backends.
  - Mint: `InitCounter=0`, `MintBatch=1`, `Burn=2`
  - Spend: `Transfer=0`, `Deliver=1`, `IncrementCounter=2`
  - Datum: `ChainOfCustody=0`, `MintCounter=1`

  Pebble assigns indices by method/variant declaration order. The canonical
  ABI is pinned at the top of both contract source files; do not reorder.

## Notes on the Pebble port

Three Pebble 0.1.8 bugs shape `incrementCounter` in particular; all have
been worked around without compromising the contract's intent. The
multi-batch tests (`incrementCounter accepts 3 new batches` and
`… rejects wrong final counter`) exercise the Bug C workaround path
(functional counter computation) and would fail any reintroduction of the
naïve `let`-accumulator pattern.

See [`pebble-known-bugs.md`](./pebble-known-bugs.md) for repros and
explanations.
